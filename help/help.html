Introduction

The PCS Marketing Request application is used to order marketing materials for various client/consultant calls and presentations. The primary functions covered are:

  • Capture required information and material requests.
  • Partition requests into distinct categories.
  • Generate previews for review and approval.
  • Upload or input prospect information.
  • Generate request forms and email to a distribution list.

The Navigation bar can be found at the top of the screen and is broken into 3 sections. Additionally, there is a cog wheel next to the user’s name which can be used to access Settings or User Preferences.

  • Home (Request)
  • Administration
  • Help
  • Settings

The Request List page is the PCS Marketing system entry point. From the Request List page you can initiate new request, preview and edit existing request, or assign request out to team members.

New Request (1)
This is where it all starts, clicking the "New Request" button in the top left corner of the grid will navigate you to the Material Request Form.
Show All (2)
The grid does not show completed items by default. This will toggle the display to include items with the status Completed.
Export (3)
Export the grid to Excel.
Search (4)
Filter Request - You may search across all columns or choose a column to filter by from the drop-down.
Action Items (5)
Quick Actions that can applied to each request.
  • Preview - Display summary or print view for the request.
  • Edit - Open the request in edit mode.
  • Mark Completed - Quickly mark a request complete, this will hide the item in the Material Request List. These items can still be viewed by toggling the “Show All” setting.
  • Assign - Quickly reassign the request from the request log screen.

The Material Request Form is where all new request originate. The form is also used for editing request.

PCS Type

The user starts by choosing a PCS type. The form will then adapt to capture the correct information for the chosen request PCS type.

  • Client - Requester will be required to search accounts and select primary account. These accounts are synced with INSITE.
  • Consultant - Requester will be required to select a consultant. These are managed in Consultant Administration.
  • Prospect - Requester will be required to select a prospect. These are managed in Prospect Administration.
  • Other - A request form without an Account, Consultant, or Prospect selection.


Account
Select Primary and Related Accounts. Use the account search to find accounts and their related accounts (synced with Insite).
Consultant
Select a consultant. Consultant list is managed in the Consultant Administration area.
Prospect
Select a prospect. Prospects list is managed in the Prospect Administration area.
Presenters
Indicate which presenters will be participating in the presentation process. The user selects a value from the left box (available) and moves it to the right box (selected). These become the selected presenters. These values can be managed in the Presenters Administration area.

Primary Materials
These are the materials that are common to all applications. The user selects a value from the left box (available) and moves it to the right box (selected). These become the selected presenters. These values can be managed in the Primary Materials area.

Performance Items
Only available for client if the user selects yes for Client Review. The user may specify dates and a reporting type of Gross, Net, or Gross & Net.
Presentation Items
User may make selections for the following properties..

  • Common Client Summary - (Yes/No)
  • Generic Cover - (Yes/No)

Presentation Sections

User specifies various standard presentation sections including products. They can then sequence the sections into the preferred order using the up or down buttons. The arrows in the middle of the control allow the user to add or remove sections in the right hand results window as needed. These values can be managed in the Presentation Section Administration area.

The primary materials that activate the Presentation Section are managed in the Primary Materials Administration area.

Custom Attribution

This section allows the user to indicate if custom attribution will be client specific. If yes then at least one period of information will be needed.

Following are the period values:

  • MTD
  • QTD
  • 3-Month
  • YTD
  • 1-Year
  • 3-Year
  • 5-Year
  • ITD

Additional Materials
The additional materials control includes a three step process. The first step is to select the group of related materials. The second step is to select an available material shown for the group (cannot be multiple selections in this column). A version (product) is selected to match with the selected material. The arrow keys are used to move the selections to the last column which will show the selected material concatenated with the selected version.

These values can be managed in the Materials Administration area.

  • Hard Copy
  • Both
Recipient
Selected from recipient list managed in the Recipient Administration area.

The Preview, Print View, and Full Screen allow user to generate previews for review, printing, and approval.

Preview

By default, the application will show previews in a modal (with a link to full screen mode in upper right corner). Users who prefer a full screen preview can change this in the User Settings.

Full Screen (1)
Display the preview in Full Screen mode. Note: for best results the request should still be printed from the printer friendly view
Forward / Backwards (2)
The forward / Backward buttons will only display if you are previewing from the Request List page. This allows the user to quickly navigate through the request list in preview mode.
Printer Friendly (3)
Will display the request in a printer friendly view and allow for nice formatted printing.
Process Request (4)
Process Request button will only be enabled if you are previewing an unprocessed request. This will validate the request and process it in the same manner as process request from the Create Request form.
Close (5)
Close the print view modal.

The grids through-out the application are consistent and offer similar functionality. The following explains the features common to grids within the PCS application.

Add New
The "Add New" button can always be found in the top left corner of the grid.
New Request
Search Filter
The "Search" filter can be found in the top right corner of each grid. By default the search will look across all columns for text matching the search criteria. Grids with many columns give the option to filter by a single column. Click the "All" button and choose a specific column to filter by.

Sort
Clicking a column header within the grid will cycle through the sorting options for the column. An arrow ▲ ▼ will be displayed next a column header with sorting applied.
  1. Ascending ▲
  2. Descending ▼
  3. Original State
Paging

The default paging size for the grids is 15 rows per page. You can click through pages using the numbered buttons at the bottom left corner of the grid.

The Page size can be changed in the settings modal. Open Settings

Actions & Context Menu

Action Icons and a Context Menu are provided to apply actions to specific records inside the grid. The Action Icons are located in the grids first column. Hovering over these icons will show you more detail about their specific function.

Right clicking on any row within the grid will show the context menu. The context menu is just another way to navigate to the same actions supplied by the action icons above.

Prospect Upload

The Prospects Management grid has a "Upload CSV" link near the Add New button in the top left corner of the grid. This can be used to upload/import prospect information into the system from a file provided by sales force.

Choose a file (1)
Click choose file button and select your CSV file to import.
Click Upload Button (2)
This will begin the upload.
Status (3)
The status will show "pending" while the file is being processed. When the file is complete it will display "Complete" or "Error".
Close Modal (4)
You may close the modal at anytime, processing will continue in the background.

User Preferences can be managed here. Open settings by clicking the cog wheel next to the users name in top right corner of screen. Note: These settings are stored in your local browser storage. this will not update settings across all browsers and computers.

Show Unsaved Changes Warning (default: yes)
Turn off the unsaved changes warning on the create request form. Navigating away from the page will automatically discard changes.
Show Full Screen Preview (default: no)
Do not show the preview modal when clicking preview. Instead user will automatically navigate to the full screen preview.
Grid Items Per Page (default: 15)
Number of items shown per page in grids across the application, large numbers may decrease page load performance.
Show Debug (default: no)
Turn on/off the debug preview link. This debug preview is used by the Information Technology team to help assist in understanding errors real-time from within application.
Default Settings
Reset the above settings to their defaults.
